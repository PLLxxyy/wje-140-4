## 项目主要需求

### 项目概述

【纯后端服务】车队调度与维护 RESTful API 服务。面向物流公司的内部管理系统提供后端接口，覆盖车辆生命周期管理、司机管理、调度派单、油耗统计、维保管理和费用核算。**本项目用于验证模型跨文件协同改动能力，禁止将多个职责合并进单一文件。**

### 核心实体（≥ 3 个，每个贯穿后端全链路）

1. **车辆（Vehicle）** — 字段：车牌号、车辆类型（LightTruck/MediumTruck/HeavyTruck/Refrigerated/Hazmat）、品牌型号、购买日期、保险到期日、年检到期日、当前状态（Available/OnTrip/Maintenance/Retired）、累计里程、油箱容量、每日固定成本
   - 关联：1辆车 → 多条调度记录、维保记录、油耗记录、费用记录
   - **贯穿后端**

2. **司机（Driver）** — 字段：姓名、手机号、身份证号、驾照类型（C1/B2/A2/A1）、驾照到期日、入职日期、当前状态（Available/OnTrip/Leave/Suspended）、月基本工资
   - 关联：1个司机 → 多条调度记录、违章记录
   - **贯穿后端**

3. **调度单（DispatchOrder）** — 字段：单号、车辆ID、司机ID、出发地、目的地、预计/实际出发/到达时间、货物描述、重量、体积、运费、油费预估、过路费预估、状态（Draft/Assigned/InProgress/Completed/Cancelled）、利润（运费-油费-过路费-人工）
   - **贯穿后端**

4. **维保记录（MaintenanceRecord）** — 字段：车辆ID、维修类型（Routine/Repair/Emergency/Inspection）、维修项目、费用、维修厂、维修日期、下次保养里程/日期、状态（Scheduled/InProgress/Completed）
   - **贯穿后端**

5. **油耗记录（FuelRecord）** — 字段：车辆ID、司机ID、调度单ID（可选）、加油量（L）、单价、总金额、当前里程、加油站、支付方式
   - **贯穿后端**

6. **费用汇总（CostSummary）** — 字段：车辆ID、统计月份、油费合计、维保合计、过路费合计、人工合计、固定成本、总成本、总收入、利润
   - **贯穿后端**

### 横切关注点（≥ 2 个）

1. **RBAC 权限控制**
   - 角色：Admin / FleetManager / Dispatcher / Driver / Mechanic
   - 触达文件层：DB role表 + RBAC中间件 + 各controller权限检查 + 类型定义

2. **审计日志**
   - 记录调度指派、状态流转、维保确认、费用核销等关键操作
   - 触达文件层：DB audit_log表 + 日志中间件 + service埋点 + 类型定义

3. **接口限流**
   - 触达文件层：限流中间件 + 配置 + Redis

### 共享枚举/常量（≥ 2 组）

1. **车辆状态 VehicleStatus**：Available / OnTrip / Maintenance / Retired
   - 定义：`backend/src/types/enums.ts`

2. **调度单状态 DispatchStatus**：Draft / Assigned / InProgress / Completed / Cancelled

3. **维修类型 MaintenanceType**：Routine / Repair / Emergency / Inspection

4. **司机状态 DriverStatus**：Available / OnTrip / Leave / Suspended

5. **支付方式 PaymentMethod**：Cash / Card / Company / ETC

### 后端中间件清单（≥ 2 个）

1. **`authMiddleware`** — JWT 鉴权
2. **`rbacMiddleware`** — 角色权限校验
3. **`auditLogMiddleware`** — 审计日志
4. **`errorHandlerMiddleware`** — 统一异常
5. **`rateLimitMiddleware`** — 接口限流
6. **`requestLoggerMiddleware`** — 请求日志
7. **`validationMiddleware`** — 请求参数校验（class-validator）

### 技术要求

- **英文短名（用于 Docker Compose）**：`fleet-api`
- **后端端口**：19210 / 3000
- **后端**：NestJS + TypeScript
- **数据库**：MySQL 8.0
- **ORM**：TypeORM
- **缓存**：Redis 7（限流 + 热点查询缓存）
- **认证**：JWT
- **API文档**：Swagger

### 部署要求

后端 + MySQL + Redis，Docker Compose 编排。
- 顶层 `name: fleet-api`
- 不写 `version:` 字段
- MySQL + Redis 命名卷 + MySQL healthcheck
- 后端 depends_on service_healthy
- `.env` 必含：`COMPOSE_PROJECT_NAME`, `DB_NAME`, `DB_USER`, `DB_PASSWORD`, `DB_ROOT_PASSWORD`, `JWT_SECRET`, `REDIS_URL`, `BACKEND_PORT`

### Git 要求

`git init` + `.gitignore`。
